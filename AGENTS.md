# Bot Commander Desktop Agent Guide

Bot Commander Desktop is an Electron application for creating and hosting Discord bots. It
supports traditional message/event commands, slash commands and buttons, BCFD response
templates, webhooks, persistent bot state, AI chat, and an agent harness that can inspect and
modify bot resources.

## Stack

- Electron and electron-vite
- TypeScript with Svelte 5
- Tailwind CSS and DaisyUI
- discord.js
- Vitest
- QuickJS for sandboxed user scripts

## Development Commands

```bash
npm install             # Install dependencies
npm run dev             # Start Electron with hot reload
npm test                # Run the Vitest suite once
npm run typecheck       # Check main/preload TypeScript and the Svelte renderer
npm run typecheck:node  # Check main and preload TypeScript only
npm run svelte-check    # Check renderer TypeScript and Svelte components only
npm run format          # Rewrite supported files with Prettier
npm run build           # Typecheck and build the application
npm run build:unpack    # Build an unpackaged application
npm run build:win       # Build the Windows package
npm run build:mac       # Build the macOS package
npm run build:linux     # Build the Linux package
npm run docs:sync       # Regenerate bundled help data from ../DiscordBots-Help
```

Tests are colocated with source as `*.test.ts`. Add or update focused Vitest coverage for behavior
changes, then run `npm test` and `npm run typecheck` before handing work off.

## Architecture

- `src/main/` is the privileged Electron process. It owns the Discord client, IPC handlers,
  persistence, AI providers and agent execution, BCFD interpretation, and QuickJS runtimes.
- `src/preload/` is the sandboxed context bridge. It exposes only explicitly whitelisted IPC
  channels to the renderer.
- `src/renderer/` is the Svelte UI. Components use Svelte 5 runes and stores for shared renderer
  state.
- `src/shared/` contains contracts and utilities used across process boundaries, including agent
  types, command codecs and patches, BCFD metadata, and linting.
- `src/main/services/bcfdLang/` contains the BCFD tokenizer, parser, and interpreter. Its
  `SPECIFICATION.md` is the in-repository language reference.
- `src/main/generated/documentationIndex.json` is generated by `npm run docs:sync`; do not edit it
  by hand.

The main process loads persisted resources from Electron's `app.getPath('userData')`. This includes
commands, interactions, settings, chats, agent sessions and memories, startup JavaScript, and bot
state.

## Project Conventions

- Follow `.editorconfig` and `.prettierrc.yaml`: two-space indentation, LF endings, single quotes,
  no semicolons, no trailing commas, and a 100-character print width.
- TypeScript strict mode is disabled, but keep new code typed and avoid introducing `any` when a
  concrete shared contract is practical.
- Use Svelte 5 runes (`$state`, `$derived`, and `$effect`) for component reactivity. Keep reusable
  renderer state in `src/renderer/src/stores/`.
- Keep privileged or filesystem/network behavior in the main process. Renderer code must call it
  through the preload bridge.
- Some domain types remain duplicated between `src/main/types/types.ts` and
  `src/renderer/src/types/types.ts`. Keep matching contracts synchronized; put genuinely shared
  process-neutral contracts in `src/shared/` when practical.
- Persist user-facing data through the existing services rather than writing directly from
  components or IPC handlers.

## Critical Workflows and Gotchas

### IPC

Electron runs with `sandbox: true`, `contextIsolation: true`, and `nodeIntegration: false`. When
adding or changing IPC:

1. Register the handler or listener under `src/main/handlers/`.
2. Add the channel to the appropriate send, invoke, or receive whitelist in
   `src/preload/index.ts`.
3. Use `window.electron.ipcRenderer` from the renderer.
4. Update `src/preload/index.d.ts` when the exposed renderer API shape changes.

### BCFD Template Language

`src/main/services/stringInfo.ts` selects the current interpreter or the legacy path according to
`settings.useLegacyInterpreter`. Preserve that compatibility unless a change explicitly removes
it.

When changing BCFD syntax or behavior, check every affected surface: the runtime interpreter,
`src/shared/bcfdLanguage.ts` autocomplete metadata, `src/shared/bcfdLint.ts`, focused tests,
`SPECIFICATION.md`, and the external help source used by `npm run docs:sync`.

Eval blocks and startup scripts execute in constrained QuickJS contexts. Do not bypass their time,
memory, stack, or host-access boundaries. `botState` persists across command executions and is
saved to user data.

### Commands and Interactions

- Traditional Discord event commands are orchestrated in `src/main/services/botService.ts`.
- Slash commands and buttons use the interaction service and slash-command registry.
- Normalize imported or externally supplied traditional commands through the shared command codec.
- Update both main and renderer contracts when changing persisted command or interaction shapes.

### AI and Agent Features

AI provider behavior is centralized in `aiProviderService.ts`. The agent harness spans agent
services in the main process, shared agent contracts, IPC whitelists and handlers, renderer stores,
and agent UI components. Preserve approval boundaries for mutating agent tools, revision checks,
cancellation, persisted sessions and memories, and streamed event handling.

Never commit credentials, bot tokens, API keys, or user data.
